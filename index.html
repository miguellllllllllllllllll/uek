Public Cloud

Was ist die Public Cloud?

Die Public Cloud ist ein Modell, bei dem IT-Ressourcen wie Server, Speicher und Anwendungen von Drittanbietern über das Internet bereitgestellt werden. Anbieter wie AWS, Microsoft Azure und Google Cloud gehören zu den bekanntesten in diesem Bereich.

Merkmale der Public Cloud

  • Skalierbarkeit: Flexible Anpassung der Ressourcen.
  • Kostenmodell: Pay-as-you-go.
  • Wartung: Vom Anbieter übernommen.
  • Zugänglichkeit: Weltweit über das Internet verfügbar.
  • Multi-Tenant-Umgebung: Mehrere Kunden teilen sich die Infrastruktur.

Vorteile der Public Cloud

  • Kosteneffizienz.
  • Schnelle Bereitstellung von Ressourcen.
  • Keine eigene Infrastruktur erforderlich.
  • Hohe Verfügbarkeit und Zuverlässigkeit.

Wichtige Git-Befehle

  • git init: Erstellt ein neues Git-Repository.
  • git clone [URL]: Klont ein vorhandenes Repository.
  • git add [Datei]: Fügt eine Datei zur Staging-Area hinzu.
  • git commit -m "Nachricht": Speichert Änderungen mit einer Beschreibung.
  • git status: Zeigt den aktuellen Status des Repositories an.
  • git log: Zeigt den Verlauf der Commits an.
  • git pull: Holt Änderungen vom Remote-Repository und integriert sie.
  • git push: Überträgt lokale Änderungen in das Remote-Repository.
  • git branch: Listet Branches auf oder erstellt einen neuen Branch.
  • git checkout [Branch]: Wechselt zu einem anderen Branch.
  • git merge [Branch]: Fügt Änderungen von einem Branch in den aktuellen ein.
  • git reset [Datei]: Entfernt eine Datei aus der Staging-Area.
  • git revert [Commit]: Macht einen Commit rückgängig.
